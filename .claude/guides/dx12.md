# DirectX 12 가이드

**적용 시점**:
- DX12 관련 기능 구현 시 (디바이스, 커맨드 큐, 리소스, PSO 등)
- DX12 리소스/배리어/동기화를 다룰 때

---

## DirectX 12 핵심 아키텍처

이 프로젝트는 DX12의 명시적 제어(Explicit Control) 모델을 따릅니다. 다음 개념들이 핵심입니다.

### 커맨드 실행 흐름

```
Command Allocator → Command List (기록) → Command Queue (제출) → GPU 실행
```

### 주요 시스템 관계

1. **Device**: 모든 리소스와 객체 생성의 팩토리
2. **Command Queue**: GPU 작업 제출 (Direct, Compute, Copy)
3. **Descriptor Heaps**: 리소스 뷰(CBV/SRV/UAV, RTV, DSV, Sampler)를 저장
4. **Pipeline State Object (PSO)**: 모든 파이프라인 상태를 하나로 묶음
5. **Root Signature**: 셰이더가 접근할 리소스 레이아웃
6. **Resource Barriers**: 리소스 상태 전환을 명시적으로 관리
7. **Fence**: CPU-GPU 동기화

자세한 내용은 `Docs/DX12_Core_Concepts.md` 참조.

---

## 디스크립터 힙 타입

- `CBV_SRV_UAV`: Constant Buffer, Shader Resource, Unordered Access (셰이더 가시적)
- `SAMPLER`: 샘플러 (셰이더 가시적)
- `RTV`: Render Target View (셰이더 비가시적)
- `DSV`: Depth Stencil View (셰이더 비가시적)

---

## 메모리 힙 타입

- `UPLOAD`: CPU → GPU (버퍼 업데이트용)
- `DEFAULT`: GPU 전용 (최고 성능)
- `READBACK`: GPU → CPU (결과 읽기용)

---

## 주의사항

- DX12는 디버깅이 어려우므로 **Debug Layer 활성화** 필수 (개발 중)
- 리소스 배리어를 잘못 설정하면 GPU 크래시 발생 가능
- 동기화(Fence)를 잘못 사용하면 데드락 발생 가능
- 멀티스레딩 구현 시 커맨드 리스트는 스레드별로 독립적이어야 함
