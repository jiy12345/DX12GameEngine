# 최적화 가이드

**적용 시점**:
- `optimize/이슈번호` 브랜치에서 작업할 때
- 성능 개선을 포함한 모든 PR 생성 시
- 벤치마크를 실행하거나 결과를 해석할 때

---

## 최적화 작업 시 필수사항

최적화 PR은 **반드시** 다음을 포함해야 합니다:

### 1. Before/After 벤치마크 결과

- FPS, 프레임 타임, GPU 타이밍 등
- `Benchmarks/` 폴더에 결과 파일 저장
- 숫자만 제시하지 말고 실행 환경(해상도, 씬 규모, 하드웨어 등) 명시

### 2. Trade-off 분석

- 메모리 사용량 변화
- 코드 복잡도 증가
- 다른 시스템에 미치는 영향
- 특정 상황에서 오히려 느려지는 경우가 있는지

### 3. 측정 방법

```bash
# 벤치마크 실행 (구현 후)
DX12GameEngine.exe --benchmark <category>
```

---

## 원칙

**최적화 없이 "성능 향상"을 주장하지 않습니다.**

- 추측만으로 "더 빠를 것이다"라고 판단하지 않음
- 반드시 수치로 증명
- 변화가 측정 오차 수준이면 "개선"이라고 부르지 않음

---

## 설계 레벨의 성능 원칙

구체적인 최적화 기법(캐시 친화성, 배치 처리, 비동기 작업 등)은 아키텍처 설계 원칙에 정의되어 있습니다.

→ `.claude/guides/architecture.md` 의 "성능 최적화 원칙" 섹션 참조
